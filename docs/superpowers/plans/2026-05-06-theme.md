# Theme Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the Math On Canvas visual style (orange accent, Geist font, light-only theme, logo) to the Starlight documentation site.

**Architecture:** Single `src/styles/theme.css` file overrides Starlight CSS variables for both `[data-theme='light']` and `[data-theme='dark']` with the same light values, neutralizing dark mode. Fonts loaded via `@fontsource-variable` packages imported inside the CSS file. Logo configured via Starlight's `logo` option in `astro.config.mjs`.

**Tech Stack:** Astro, Starlight 0.38.x, CSS custom properties, `@fontsource-variable/geist`, `@fontsource-variable/geist-mono`

- [ ] **Step 3: If anything looks wrong**

Common issues and fixes:
- **Font not loading:** Check DevTools Network tab for `@fontsource-variable/geist` — if 404, verify package is installed and `@import` path is correct in `theme.css`.
- **Colors not overriding:** Check DevTools → Elements → `<html>` — confirm `data-theme="light"` attribute is set. If `data-theme="dark"`, the dark block should still apply light values. Check specificity — Starlight's props.css may need `!important` on accent if it's more specific.
- **Logo not showing:** Verify `src/assets/logo.svg` exists and the path in `astro.config.mjs` is `'./src/assets/logo.svg'`.
